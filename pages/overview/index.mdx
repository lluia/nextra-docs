## NextAuth.js â†’ Auth.js

NextAuth.js has been re-architected and rebranded as **Auth.js**. NextAuth.js was initially born as an authentication library for [Next.js](https://nextjs.org/). With v5 we took a step back and refactored the whole library to be an authentication layer which is framework agnostic.

<img
  src="/img/etc/authjs-architecture.png"
  width="500"
  className="mx-auto mt-4"
/>

Auth.js stays as a core JS authentication layer which can be used in any
framework (using the specific package for the framework). [Have a look at our history](/overview/contributors#history) on why NextAuth.js was rebranded to Auth.js.

  Migrating from v4? See [our migration guide](/migrating-to-v5).
</Callout>

## Authentication methods

There are 3 ways to authenticate users with Auth.js:

- [OAuth authentication](/getting-started/providers/oauth-tutorial) ( _Sign in with Google, Github, LinkedIn, etc..._ )
- [Paswordless authentication](/getting-started/providers/email-tutorial) ( _Magic links, OTP codes, WebAuthn, etc..._ )
- [External Auth](/getting-started/providers/credentials-tutorial) ( _Integrating with external APIs, etc..._ )

Additionally, Auth.js can be integrated with an external database via Database adapters, in case you need or want to store user data.
